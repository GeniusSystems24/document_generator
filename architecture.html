PHASE 1 · ENGINE

واجهة انسيابية تبني المستند

نفس الاستدعاءات تتقدّم بالموضع تلقائياً، وتُنشئ صفحات جديدة عند الحاجة، وتطبع باللغتين — والتحقّق يسبق التوليد دائماً.

invoice.builder.ts 
const doc = new GeniusPdfBuilder({ dir: 'rtl', locale: 'ar' });

doc
  .addReportHeader({ company, vat, logo })
  .addInfoBox({ label: 'العميل', value: client })
  .addGrid(items, {
    preset: 'elegant',
    totals: ['sum'],          // صف مجموع تلقائي
  })
  .addSummary({ vat: 0.15, discount: 0.05 })
  .addQRCode(zatca(invoice), { align: 'start' });

// التحقّق المالي يسبق التوليد دائماً
await doc.validate().then(d => d.save());

PHASE 0 · FOUNDATION

التأسيس قبل أيّ مكوّن

رموز تصميم موحّدة، طبقة i18n بلا نصوص ثابتة، هيكل تطبيق، ومعرض مكوّنات حيّ — كلّها أُرسيت في المرحلة صفر.

design-tokens · colors · spacing · type

رموز التصميم

--ink --teal --paper --brass --jade --rose

مصدر واحد للحقيقة للألوان والمسافات والخطوط — تلتزم به كل الصفحات والقوالب بلا قيم صلبة.

i18n · AR / EN · RTL ↔ LTR

طبقة التدويل

t('invoice.total')
ar →الإجمالي المستحق
en →Total due

لا نصّ ثابت داخل أيّ مكوّن — كلّ سلسلة تمرّ بـ i18n، والاتجاه يُشتقّ من اللغة فيقلب الواجهة والمستند معاً.

CLEAN ARCHITECTURE · MVC-ON-REACT

طبقات يتّجه اعتمادها للداخل

الـ View بلا منطق، والـ Controller (hooks + use cases) يحمل المنطق، والـ Model في الـ domain. الاعتماد يسير من الخارج نحو القلب فقط.

domain
قلب المكتبة — كيانات، عقود مستودعات، و use cases قابلة للاختبار باستقلال عن React.
Model + Logic
data
تنفيذ المستودعات — مصادر بيانات ومحوّلات تُترجم بين الخارج والـ domain.
Data access
infrastructure
محوّلات خارجية — محرّك الـ PDF، التصدير، التخزين، الطباعة، المشاركة، والباركود.
Adapters
presentation
View + Controller — مكوّنات وشاشات، و hooks/viewmodels تربط الـ use cases بالعرض.
View + Controller
templates
قوالب جاهزة — reports / financial / sales / hr / vouchers، مبنيّة من المكوّنات.
Composition

هيكل المجلدات · src/

الخريطة على القرص

src/ core/ # config · constants · utils · i18n · design-tokens · types domain/ # entities · repositories(interfaces) · usecases ← Model + logic data/ # repositories(impl) · datasources · mappers infrastructure/ # pdf-engine · export · storage · print · share · barcode presentation/ # View + Controller components/ # common/ + pdf/ (reusable) pages/ # screens hooks/ # controllers (bind use cases to View) viewmodels/ # map domain data to display layouts/ templates/ # reports/ financial/ sales/ hr/ vouchers/

قائمة التحقّق الختامية

المعايير التي يُراجَع بها المشروع كاملاً بعد آخر مرحلة.

✓فصل صارم: View بلا منطق، Controller (hooks/usecases) يحمل المنطق، Model في domain.
✓اتجاه الاعتماد للداخل فقط (presentation → domain، لا العكس).
✓كل مكوّن PDF قابل لإعادة الاستخدام وموجود في المعرض بمثالي AR/EN.
✓لا نصوص ثابتة؛ كلها عبر i18n.
✓RTL/LTR يعمل في كل الشاشات والقوالب.
✓التزام design tokens بلا قيم صلبة.
✓التحقّق المالي يمنع التوليد عند وجود خطأ.
